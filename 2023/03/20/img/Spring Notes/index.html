Spring

1.Spring简介

1.1 Spring概述

官网地址: https://spring.io/projects/spring-framework

Spring 是最受欢迎的企业级 Java 应用程序开发框架，数以百万的来自世界各地的开发人员使用Spring 框架来创建性能好、易于测试、可重用的代码。

Spring 框架是一个开源的 Java 平台，它最初是由 Rod Johnson 编写的，并且于 2003 年 6 月首次在 Apache 2.0 许可下发布。

Spring 是轻量级的框架，其基础版本只有 2 MB 左右的大小。

Spring 框架的核心特性是可以用于开发任何 Java 应用程序，但是在 Java EE 平台上构建 web 应用程序是需要扩展的。Spring 框架的目标是使 J2EE 开发变得更容易使用，通过启用基于 POJO编程模型来促进良好的编程实践。

项目列表：https://spring.io/projects

1.3 Spring Framework

Spring的基础设施

1.3.1 Spring Framework特性

非侵入式: 不破坏程序原有结构

控制反转: IOC–Inversion of Control

面向切面编程: AOP– Aspect Oriented Programming

容器: Sprng IOC是一个容器负责管理组件对象的生命周期

组件化:Spring实现了使用简单的组建配置组合成一个复杂的应用

声明式:对编程式的一种简化

一站式:可以整合各种开源框架和第三方库

2. IOC

2.1 IOC容器

2.1.1 IOC思想

对象生命周期的管理不再由程序员决定而由容器进行管理

依赖注入是IOC的一种具体实现

2.1.2 IOC容器在Spring中的实现

Spring的IOC容器就是IOC思想的一个落地的产品实现。IOC容器中管理的组件也叫做bean。在创建bean之前，首先需要创建IOC容器。Spring 提供了IOC容器的两种实现。

1.BeanFactory

2.ApplicationContext

2.2.2 bean对象的获取

可以通过ApplicationContext的实现类的对象的getBean方法获取，getBean方法重载了如下的几种函数签名

Modifier and Type Method
T getBean(Class requiredType)
T getBean(Class requiredType, Object… args)
Object getBean(String name)
T getBean(String name, lass requiredType)
T getBean(String name, Object… args)

根据类型来获取bean时，在满足bean唯一性的前提下，其实只是看：『对象 instanceof 指定的类型』的返回结果，只要返回的是true就可以认定为和类型匹配，能够获取到。

2.2.4 自动装配：

根据指定的策略，在IOC容器中匹配某个bean，自动喂bean中实体类类型的属性赋值

可以通过bean标签中的autowire属性设置装配策略

自动装配的策略有：

no，default: 不开启自动装配

byType: 注意如果根据类型没有找到任何一个类型匹配的bean，此时不装配，若根据类型可以找到不唯一的bean，则throw一个NoUniqueSuchBeanDefinition的Exception

byName: 以要赋值的字段名作为id寻找bean
